Developer API Docs

RESTful API endpoints with structured meta input/output format for integration.

Base URL & Health

All API endpoints are relative to your server base URL.

Base URL: http://localhost:5000/api
GET /api/health System health check & model status

Authentication Endpoints

JWT token-based authentication. Include token in Authorization: Bearer <token> header.

POST /api/auth/register Create a new user account
POST /api/auth/login Authenticate and receive JWT token
POST /api/auth/logout Clear server session
GET /api/auth/me AUTH Get current user profile

Prediction API

GET /api/symptoms List all 132 symptoms with severity
GET /api/diseases List all 41 predictable diseases
POST /api/predict Predict disease from symptoms
GET /api/predictions/history AUTH User's prediction history

Meta Input Format

{
  "symptoms": ["headache", "high_fever", "cough"],
  "patient_info": {
    "age": 30,
    "gender": "male"
  }
}

Meta Output Format

{
  "success": true,
  "prediction": {
    "disease": "Typhoid",
    "confidence": 85.5,
    "severity_score": 65.2,
    "description": "An acute illness...",
    "precautions": ["..."],
    "medications": ["..."],
    "diets": ["..."],
    "workouts": ["..."]
  },
  "meta": {
    "model": "Support Vector Machine (Linear Kernel)",
    "symptoms_analyzed": 3,
    "timestamp": "2026-04-20T12:00:01"
  }
}

Quick Start Example

# Register a user
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"John","email":"john@test.com","password":"secret123"}'

# Predict disease
curl -X POST http://localhost:5000/api/predict \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{"symptoms":["headache","high_fever","nausea"]}'
